How it works

PU Prime Captions runs a complete pipeline — from raw media to a fully subtitled video — with no manual intervention.

📥

Input

Local file, folder, or URL (YouTube, Twitch…)

›
›
🎙

Transcription

Whisper / WhisperX with voice activity detection (Silero or Pyannote VAD)

›
›
🌐

Translation

Google Translate or Gemini AI with context-aware chunking

›
›
🎬

Export

.srt, softsub (MP4 + subtitle track) and hardsub (burned-in captions)

🎯

Word-level alignment

WhisperX aligns every word to the exact moment it is spoken, producing perfectly synchronized subtitles.

⚡

GPU acceleration

Auto-detects CUDA, MPS, or CPU. Supports NVENC, VAAPI, AMF and other hardware encoders via FFmpeg.

📦

Automatic download

Pass a YouTube, Twitch, or any yt-dlp-supported URL directly. PU Prime Captions downloads and processes it automatically.

🔤

Softsub & Hardsub

Outputs an MP4 with an embedded subtitle track (softsub, optional) and a separate video with captions burned into the frames (hardsub).

📝

TLTW summary

"Too Long To Watch" — generates a Markdown summary with title, tags, key points, and timestamped chapters using Gemini AI.

🐳

Docker support

Run inside an isolated container with GPU support (NVIDIA Container Toolkit). No host dependencies beyond Docker itself.

Transcription Models

PU Prime Captions uses OpenAI's Whisper models. Pick the right balance between speed and accuracy.

Model Parameters VRAM Speed Quality Best for
tiny 39 M ~1 GB ●●●●● ●○○○○ Quick tests
small 244 M ~2 GB ●●●●○ ●●○○○ Low VRAM machines
medium 769 M ~5 GB ●●●○○ ●●●○○ General use
turbo 809 M ~6 GB ●●●●○ ●●●●○ Great speed/quality tradeoff
large-v3 1.55 B ~10 GB ●●○○○ ●●●●○ High accuracy
large-v3-turbo default 809 M ~6 GB ●●●●○ ●●●●● Best accuracy + speed
distil-large-v3 756 M ~6 GB ●●●●○ ●●●○○ English, fast and lightweight

VRAM estimated for float16. Actual usage depends on batch size and audio length.

Using on Google Colab

No GPU? Use Google's free infrastructure without installing anything on your computer.

  1. 1

    Open the Notebook

    Click the button below. The notebook will open directly in Google Colab (a Google account is required).

  2. 2

    Run the Prepare cell

    Installs PU Prime Captions, FFmpeg, fonts, and all dependencies. Mount Google Drive to access your files.

  3. 3

    Set options in the Configure cell

    Set the input folder, Whisper model, target language for translation, video codec and other options — all through a visual form.

  4. 4

    Run the Run cell

    Processing starts. Follow the live log. Generated files are saved directly to your Google Drive.

⚠ Important tips

  • › Use the GPU T4 runtime (free) for faster transcription.
  • › Upload your videos to the configured Google Drive folder before running.
  • › You can pass a YouTube URL directly in the input_path field.
  • › To use Gemini (translation/TLTW), get a free API key at aistudio.google.com.
  • › Free Colab sessions disconnect after ~12 hours.

Third-party tools

PU Prime Captions integrates the best open-source tools in ML and video processing.

🔊

Whisper

OpenAI

Speech recognition (ASR) model trained on 680,000 hours of multilingual audio. The foundation of ppcaptions transcription.

⚡

WhisperX

m-bain (ppcaptions fork)

Accelerated Whisper fork with forced word-level alignment and VAD (voice activity detection) support. Default transcription engine.

🎞

FFmpeg

FFmpeg Team

Video and audio processing engine. Handles subtitle muxing (softsub), hardsub burning, and hardware-accelerated transcoding.

📥

yt-dlp

yt-dlp team

Video downloader supporting YouTube, Twitch, Vimeo and thousands of other platforms. Integrated directly into ppcaptions for URL inputs.

🔥

PyTorch

Meta AI

Deep learning framework that runs the Whisper models. Supports CUDA (NVIDIA), MPS (Apple Silicon), and CPU with automatic detection.

✨

Gemini AI

Google DeepMind

Optionally used for high-quality translation and for generating TLTW (Too Long To Watch) summaries with chapters and key points.

🎤

Silero VAD

Silero

Default voice activity detector (VAD) used by WhisperX. Strips silences and improves subtitle segmentation automatically.

🌍

deep-translator

nidhaloff

Python library that abstracts multiple translation APIs. Used to integrate Google Translate with context-aware block chunking.

📄

pysrt

byroot

.srt file manipulation: reading, writing, timestamp editing, and intelligent subtitle block segmentation.

Local installation

Prefer to run it on your own machine? Choose an installation method.

Install uv and run:

uv tool install git+https://github.com/xvictorlopes/ppcaptions.git

To upgrade:

uv tool upgrade ppcaptions

To use:

ppcaptions -i /path/to/video.mp4
pip install ppcaptions

Also install FFmpeg for your platform (see the Source code tab). Then run: 

ppcaptions -i /path/to/video.mp4

Requirements: Python 3.9–3.12, Git, FFmpeg.

git clone https://github.com/xvictorlopes/ppcaptions.git
cd ppcaptions
pip install -r requirements.txt --upgrade

To run:

python3 ppcaptions.py -i /path/to/video.mp4

Windows: install FFmpeg via choco install ffmpeg-full. Linux: apt install ffmpeg.

# Build
docker compose build

# Process a video
docker compose run --rm ppcaptions -i /data/video.mp4 --translate pt

# With GPU (NVIDIA Container Toolkit required)
docker compose run --rm --gpus all ppcaptions -i /data/video.mp4

Place your files in ./data or mount a different volume.

CLI Reference

All available options in the PU Prime Captions CLI.

ppcaptions -i [input_path] [options]

Path to a media folder, single file, or URL (YouTube, etc.). When a URL is provided, yt-dlp downloads the content before processing.

ppcaptions -i /videos/
ppcaptions -i https://www.youtube.com/watch?v=…

Transcription engine. whisperx (default) or whisper. WhisperX is faster and aligns words individually.

Name or path of the Whisper model. Default: large-v3-turbo.
Options: tiny, small, medium, large, large-v3, turbo, large-v3-turbo, distil-large-v2, distil-large-v3, and others.

Execution device: auto (default), cpu, cuda. In auto mode, prefers CUDA > MPS > CPU.

Target language code. If the same as the source language, translation is skipped.
ppcaptions -i /videos/ --translate pt-BR
ppcaptions -i /videos/ --translate fr

google (default) or gemini. With Gemini, provide the key via --gemini_api_key. Multiple keys can be supplied for quota rotation.

Enables "Too Long To Watch" summary generation with Gemini. Requires --gemini_api_key. Outputs a Markdown file with title, tags, key points, and timestamped chapters.
Use --output_tltw to set the output folder for summaries.

Default video: h264. With hardware acceleration: h264_nvenc, h264_vaapi, hevc_nvenc, etc.
Default audio: aac. Options: libopus, libmp3lame, pcm_s16le.

Disables softsub generation (subtitle track in the MP4 container) or hardsub generation (burned-in captions). Both are enabled by default.
Use --subtitle_formats srt,txt to control which subtitle file formats are exported.

Processes .srt files found in the input folder (translation / TLTW) without re-transcribing the audio. If a .srt has the same name as a video, it will be used instead of transcription.
